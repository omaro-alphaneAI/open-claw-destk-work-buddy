# WorkBuddy Pet 用户操作指南

[English Version](../README.en.md)

## 1. 这是什么

WorkBuddy Pet 是一个桌宠应用。

它有两种使用方式：

- `Cute Mode`
  不联网，只在本机陪伴、提醒、卖萌。
- `WorkBuddy Mode`
  只有当你输入自己的 `API URL`、`Model`、`API Key` 后，桌宠才会连接你的服务，变成你的个人工作伙伴。

## 2. 怎么安装

推荐直接从 GitHub `Releases` 下载，不需要本地安装 Node。

按平台下载：

- macOS：下载 `*.dmg`
- Windows：下载 `*.exe`
- Linux：下载 `*.AppImage` 或 `*.deb`

安装完成后，直接打开应用即可。

## 3. 第一次启动会发生什么

第一次打开应用时：

- 默认进入 `Cute Mode`
- 默认不联网
- 默认不共享本机上下文
- 不需要 API key 也能直接使用

你可以先观察桌宠动作、打开聊天抽屉、打开工作台，再决定要不要接入自己的 WorkBuddy。

## 4. 如何切换到 WorkBuddy Mode

1. 打开应用工作台
2. 打开 `WorkBuddy 模式`
3. 填写以下内容：
   - 服务名称
   - API URL
   - 模型名
   - API key
4. 保存 API key
5. 开始在聊天框输入你的问题

## 5. API URL 应该怎么填

当前支持 OpenAI-compatible 接口：

- `Chat Completions`
  例如 `/v1/chat/completions`
- `Responses API`
  例如 `/v1/responses`

你可以让协议保持 `自动识别`，应用会根据 URL 自动判断。

## 6. 安全限制有哪些

为了避免误泄漏，本应用对 WorkBuddy 出站请求做了限制：

- 默认只允许 `HTTPS`
- 只有 `localhost / 127.0.0.1 / ::1` 这类本机地址允许使用 `HTTP`
- 不允许跟随 3xx 重定向
- 不允许在 `API URL` 中内嵌账号和密码
- 不允许在 `API URL` 中放 `token`、`api_key`、`password` 之类的敏感 query 参数

如果你的配置不符合这些规则，应用会直接拦截，而不是悄悄发送请求。

## 7. 默认不会上传什么

默认不会上传到远端的内容包括：

- 本机路径
- 本机文件内容
- 前台窗口标题和应用名
- 代码改动快照
- 环境变量
- 剪贴板
- 用户目录信息
- 本地身份描述文件

只有你手动打开相关开关时，才会允许额外共享有限的上下文。

## 8. “允许显式共享上下文”是什么意思

这个开关打开后，应用会把少量你当前会话相关的摘要拼进系统提示里，例如：

- 当前意图摘要
- 最近几条环境提示摘要

这些内容在进入请求前会先做本地脱敏。常见的：

- 本地路径
- 明显的 token / API key
- `Bearer ...`

会先被替换掉。

## 9. API key 会存在哪里

- macOS / Windows
  优先使用系统安全存储能力
- Linux
  如果运行环境没有安全持久化能力，则只保留在当前会话里；关闭应用后需要重新输入

## 10. 如何导入自己的宠物图片

1. 打开工作台
2. 点击 `导入宠物图片`
3. 选择你的图片文件
4. 导入完成后，点击 `用我的宠物`

支持常见图片格式：

- `png`
- `jpg`
- `jpeg`
- `webp`
- `gif`

## 11. 日常使用建议

- 只想陪伴：保持 `Cute Mode`
- 想接入自己的 AI：切到 `WorkBuddy Mode`
- 想尽量严格：不要打开任何额外共享开关
- 想避免误发：不要把你不想出机的内容直接输入到聊天框

## 12. 常见问题

### 12.1 为什么填了 API 还是不能回复

先检查：

- `API URL` 是否正确
- 模型名是否正确
- API key 是否已保存
- 当前 URL 是否符合安全限制

### 12.2 为什么 Linux 上重启后 API key 没了

这通常说明当前 Linux 运行环境没有可用的安全持久化后端，所以应用按设计退回到了“仅当前会话保存”。

### 12.3 为什么我看到了“已拦截”

这表示应用主动拦截了一个不安全或不合规的 WorkBuddy 配置，常见原因包括：

- 使用了远端 `HTTP`
- URL 里内嵌了账号密码
- URL 带了敏感 query 参数

### 12.4 不填 API 能不能用

可以。那就是 `Cute Mode`，不会联网。

## 13. 平台说明

- macOS
  功能最完整，支持系统语音播报和前台窗口探测
- Windows
  可直接作为桌宠和 WorkBuddy 使用
- Linux
  可直接作为桌宠和 WorkBuddy 使用，但 API key 是否能持久保存取决于系统安全存储能力
